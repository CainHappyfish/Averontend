# Vue 实战 · 第二章：ManageSystemDemo 进阶改造（B 路线）

## 章节定位

本章是第一章的进阶版，适合已经跑通基础闭环的同学。

- A 路线（保底）：跑通登录 -> 主布局 -> 列表页
- B 路线（进阶）：在不重写项目的前提下，做“可维护性提升”

## 本章目标

围绕已有项目，完成三类改造：

1. 请求层统一（拦截器、错误处理、类型约束）
2. 权限与路由体验优化（守卫、兜底页、无权限提示）
3. 页面可维护性改造（组件拆分、状态归位、复用逻辑）

## 推荐改造顺序

### 1) 请求层收敛

- 统一封装 `request`（baseURL、超时、错误提示）
- 区分业务错误与网络错误
- 给关键接口补返回类型

### 2) 登录态与权限

- token 无效时统一清理并跳登录
- 未授权路由增加友好提示页
- 菜单与路由权限策略保持一致

### 3) 页面结构与复用

- 把过长页面拆为“容器组件 + 展示组件”
- 重复逻辑提为 composable（如筛选、分页、表单提交）
- 保留最小可测试单元

## 本章验收标准

- [ ] 同一个错误场景不会在多个页面重复处理
- [ ] 权限异常路径可预测（过期、无权限、未登录）
- [ ] 目标页面代码复杂度明显下降（职责更清晰）
- [ ] 你能用 10 分钟讲清楚“改造前后差异”

## 建议提交节奏

1. `refactor: unify request layer and api error handling`
2. `feat: improve auth guard and unauthorized fallback`
3. `refactor: split page into composables and reusable components`

- 有经验同学：2~4 小时
