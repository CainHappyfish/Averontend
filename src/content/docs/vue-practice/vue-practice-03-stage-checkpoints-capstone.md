# 阶段关卡与毕业项目（从 0 到可交付）

## 使用说明

这一章是整套课的阶段验收，不是新知识灌输。建议在以下节点使用：

- 完成基础三节后做关卡 A
- 完成 JS 提高与工程化后做关卡 B
- 完成 TS + Vue + Node 后做关卡 C
- 最后完成毕业项目

## 关卡 A：静态页与交互基础

目标：独立完成一个“可点击、可展示信息”的个人卡片页。

验收：

- [ ] 结构语义化（标题、段落、链接）
- [ ] 有基础样式层次
- [ ] 有至少一个事件交互

## 关卡 B：工程化与接口意识

目标：能独立拉起项目并说清脚本和构建流程。

验收：

- [ ] 能解释 `npm run dev` 与 `npm run build`
- [ ] 能排查至少 1 个环境/端口类错误
- [ ] 能读取一个接口请求并定位状态码问题

## 关卡 C：Vue + TS + Node 综合

目标：完成一个最小管理系统闭环。

验收：

- [ ] 登录态与路由守卫闭环
- [ ] 列表页 + 新增/编辑/删除中至少两项可用
- [ ] 至少一处类型约束和一处请求封装

## 毕业项目要求

请实现一个你自己的“轻管理后台”，题材自选（如任务管理、读书清单、社团排练、库存管理）。

最低要求：

- 前端：Vue 3 + TypeScript
- 后端：Node/Koa 或 Mock（二选一）
- 功能：登录（可简化）+ 列表查询 + 新增 + 编辑 + 删除
- 交付：README（运行步骤、功能说明、已知问题）

## 评分建议（自评/互评）

- 功能完整度：40%
- 代码可维护性：30%
- 文档与表达：20%
- 稳定性与边界处理：10%

## 卡住时优先级

如果时间不够，优先保证：

1. 可运行
2. 核心闭环
3. 文档可复现

做到这三点，就已经是合格交付。
