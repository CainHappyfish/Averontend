# Vue 实战 · 第一章（A 路线）：冰川日菜的练习室管理系统

## 章节定位

本章要做的是一个前端层面的 CRUD 管理系统。它不追求复杂后端能力，也不要求接入真实业务接口，而是把重点放在“管理系统应该怎么组织页面、数据和交互”上。

这是 A 路线（保底路线）：先打通最小可运行闭环。后续你可以进入：

- 第二章（B 路线）：进阶改造（请求层、权限、复用）
- 第三章：阶段关卡与毕业项目

日菜突然觉得“练习安排、曲目进度、成员分工全靠脑袋记，好像一点都不噜～♪”，于是把这个任务交给你：做一个轻量的乐队练习管理后台，帮她把练习计划、成员安排、曲目清单和练习反馈都整理得清清楚楚。

---

## 要做什么

你需要完成一个有后台管理系统结构的前端项目，核心是实现一组可演示的增删改查功能。

系统至少包含：

- 一个登录页，用来进入管理系统
- 一个主布局，包含侧边栏、顶部栏和内容区
- 一个列表页，用来展示练习记录或曲目记录
- 一个新增表单，用来添加新的练习安排
- 一个编辑表单，用来修改已有记录
- 一个删除操作，用来移除不需要的记录
- 一个详情或查看入口，用来快速浏览单条记录

数据可以先放在前端本地，例如数组、`localStorage` 或 mock 数据文件。现阶段不需要真正连接数据库，也不需要实现完整权限系统。

---

## 业务设定

这个管理系统可以围绕“日菜的练习室”展开。页面里的数据不必是普通的用户、订单、商品，而是更贴近角色背景的内容。

可以管理的对象包括：

- 练习计划：记录练习日期、时间、地点、主题和状态
- 曲目清单：记录曲名、难度、负责成员、练习进度
- 成员安排：记录成员姓名、乐器位置、是否参加本次练习
- 练习反馈：记录日菜对本次练习的评价，例如“很有趣”“还可以更噜～”

只要能完整体现“列表展示 -> 新增 -> 编辑 -> 删除 -> 查看结果变化”这条链路即可。

---

## 页面建议

页面可以按下面的方式组织：

1. 登录页：输入任意账号密码后进入系统
2. 首页：展示今天的练习数量、待确认曲目、已完成练习等概览信息
3. 练习管理页：实现主要 CRUD 功能
4. 曲目管理页：作为扩展 CRUD 页面，可选完成
5. 关于页面：简单介绍这是一个以冰川日菜为背景的前端练习项目

界面风格可以使用清爽明亮的蓝绿色，呼应冰川日菜的角色印象。文案可以轻松一点，但功能结构仍然要像一个真正的管理系统。

---

## 完成标准

完成后，你应该能演示下面这条流程：

1. 打开项目并进入登录页
2. 登录后进入管理后台
3. 在练习管理页看到已有练习记录
4. 新增一条练习安排
5. 编辑这条练习安排
6. 删除一条不需要的记录
7. 刷新页面后，数据表现符合你的设计

本章的重点不是把功能做得很大，而是先把一个前端 CRUD 管理系统的基本骨架做完整。只要页面结构清楚、数据变化可见、操作流程顺畅，就达到了这一章的练习目标。
