# 预备章 02：终端、目录与命令行

## 本节目标

学会最常用的终端操作，不再害怕“黑框框”。

完成标准：

- 能定位当前目录、切换目录、查看文件
- 能运行并理解常见 npm 脚本
- 知道命令失败时如何读取错误关键信息

## 三个必须会的命令

```bash
pwd
ls
cd <目录名>
```

含义：

- `pwd`：当前在哪个目录
- `ls`：当前目录有哪些文件
- `cd`：切换到目标目录

## 跑脚本前先看 package.json

新手常见问题是“命令瞎试”。正确顺序：

1. 先看 `package.json` 里的 `scripts`
2. 再执行对应命令，例如 `npm run dev`
3. 看报错是否来自脚本本身还是环境问题

## 看懂报错的最小方法

先抓 3 个关键信息：

- 错误代码（例如 `EADDRINUSE`）
- 报错模块（例如 `vite` / `node`）
- 触发位置（命令行哪一步报错）

不要一上来全贴日志，先总结“错误代码 + 发生动作 + 你已尝试操作”。
