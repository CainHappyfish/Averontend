# 预备章 01：环境与工具就位（零基础友好）

## 本节目标

在开始 HTML/CSS/JS 之前，先把学习环境搭好，避免“内容会了但跑不起来”。

完成标准：

- 知道前端学习需要的最小工具链
- 本机能打开项目并运行最小页面
- 知道遇到报错时先看哪里

## 必装工具清单

- 编辑器：`Cursor` 或 `VS Code`
- 浏览器：`Chrome`（用于 DevTools）
- Node.js：建议 LTS（本项目建议 `Node 22+`）
- Git：用于拉代码与版本管理

> 新手建议：先不纠结插件大全，只装基础工具，边学边补。

## 10 分钟完成环境验证

在终端执行：

```bash
node -v
npm -v
git --version
```

如果都能输出版本号，说明基础环境正常。

## 第一次跑起前端项目

```bash
npm install
npm run dev
```

看到本地地址（如 `http://localhost:5173`）并能打开页面，算通过。

## 常见报错快速处理

- `command not found`：工具未安装或 PATH 未生效
- `EADDRINUSE`：端口被占用，换端口或结束旧进程
- 下载依赖很慢：临时换 npm 镜像源再安装
- 版本不匹配：按项目要求切换 Node 版本
