# 预备章 03：浏览器调试与“卡住时流程”

## 本节目标

掌握前端学习最重要的保命技能：遇到问题时有固定排查流程。

## DevTools 必会区域

- `Elements`：看 DOM 结构与样式覆盖
- `Console`：看 JS 报错与日志
- `Network`：看请求是否成功、响应内容是否正确

## 新手卡住时的 5 步流程

1. **复现问题**：确保问题可稳定出现
2. **缩小范围**：判断是页面结构、样式、脚本还是接口问题
3. **看报错正文**：不要只看最后一行
4. **最小改动验证**：一次只改一个点
5. **记录结论**：写下“原因-修复-验证结果”

## 本课程统一的卡关处理建议

- 单点卡住超过 20 分钟：先回退到“上一可运行状态”
- 连续两次尝试失败：去文档对应小节重看核心概念
- 仍无法解决：整理问题描述后再提问（附关键日志）

## 提问模板（建议直接复制）

```text
我在做：<课程名 + 任务>
我执行了：<命令或操作>
我期望：<预期结果>
实际发生：<实际结果>
错误关键信息：<error code / 关键日志>
我已尝试：<尝试过的1~2种方式>
```
