# TypeScript 入门：特性、元组、枚举与 never

`TypeScript` 是 `JavaScript` 的**类型超集**，支持 `ES6` 语法，并支持类、接口、继承、泛型等面向对象与类型系统的概念。

`TypeScript` 的特性主要有如下：

- **类型批注和编译时类型检查**：在编译时批注变量类型
- **类型推断**：没有批注时也会自动推断变量类型
- **类型擦除**：编译为 JS 时，**类型信息**在运行时不复存在（以编译目标为准）
- **接口**：用接口来定义对象类型
- **枚举**：将取值限定在一组友好名字中
- **Mixin**、**泛型**、**名字空间**、**元组** 等
- …

## 元组（tuple）

允许表示一个**已知元素数量**和**各位置类型**的数组，**各元素类型不必相同**。

```tsx
let tupleArr: [number, string, boolean]
tupleArr = [12, '34', true] // ok
// tupleArr = [12, '34'] // 不行：缺少第三项或长度不对
```

## 枚举（enum）

`enum` 类型是对标准数据类型的一个补充，可为一组数值赋予**友好名字**。

```tsx
enum Color {
  Red,
  Green,
  Blue,
}
let c: Color = Color.Green
```

若某一成员**赋值为字符串字面量**，则**后续**成员在**不指定 initializer** 时**不能**再依赖数字自增，需要**手写字面值**或恢复数字自增，否则会报错，例如：

```tsx
enum Direction {
  Up = 'UP',
  // Down, Left, Right // error: Enum member must have initializer
}
```

多次声明**同名** `enum` 时，在**同一作用域/合并**语义下**成员**可能**叠加上去**（以 tsconfig 与合并规则为准，日常避免依赖「重复声明」）。

## `never`

`never` 是**底类型**的一种直觉表述：**不可达/不存在** 的值。多用于「**必然抛错** / **死循环不返回**」的函数**返回位置**，在控制流分析里也会用来表示「**穷尽**」了联合。

```tsx
let a: never
// a = 123 // 错误

a = (() => {
  throw new Error('错误')
})()

function error(message: string): never {
  throw new Error(message)
}
```

> 与「可赋值性」的严格表述以 TypeScript 手册/规范为准，面试**概念**上记住：`never` 在子类型关系里**很特殊**，且在联合/`switch` 穷尽检查里常见。
